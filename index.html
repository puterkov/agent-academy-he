רמות סיכון לכלים

ירוק

לסכם, לחלץ, לסווג, להציע, להכין טיוטה.

צהוב

להוסיף שורה לגיליון, לעדכן סטטוס, ליצור טיוטת אירוע.

אדום

לשלוח מייל, למחוק, לקבוע פגישה, לשנות נתוני סטודנטים או כסף.

מילון מונחים

המונחים שאתה צריך להכיר — בלי הנדסת תוכנה כבדה

Agent / סוכן

מה זה? מודל שקיבל תפקיד, הוראות, כלים, זיכרון וגבולות פעולה.

טעות נפוצה: לחשוב שסוכן הוא פשוט צ׳אט חכם.

Tool / כלי

מה זה? פעולה שהסוכן יכול להפעיל: חיפוש מייל, הוספת משימה, יצירת טיוטה.

שאלת בדיקה: האם הכלי קורא בלבד או גם משנה משהו?

API

מה זה? דלת כניסה למערכת אחרת. דרך שבה הסוכן מבקש ממערכת לעשות פעולה.

דוגמה: הוספת שורה ל־Google Sheets.

JSON / פלט מובנה

מה זה? דרך מסודרת להחזיר מידע בשדות קבועים שאפשר להעביר למערכת אחרת.

טעות נפוצה: להסתפק בסיכום יפה במקום מידע שאפשר לפעול עליו.

RAG

מה זה? הסוכן מחפש קודם במסמכים שלך ואז עונה על בסיסם.

דוגמה: שאלה על תקנון, סילבוס או נוהל מוסדי.

Guardrails / מעקות בטיחות

מה זה? כללים שמונעים מהסוכן לבצע פעולות מסוכנות או להחזיר פלט לא תקין.

דוגמה: אסור לשלוח מייל בלי אישור מפורש.

Human in the Loop

מה זה? האדם מאשר בנקודות קריטיות לפני פעולה אמיתית.

דוגמה: הסוכן מכין טיוטת מייל, מנשה מאשר, ורק אז שולחים.

Prompt Injection

מה זה? טקסט חיצוני שמנסה לגרום לסוכן להתעלם מההוראות שלו.

כלל: מיילים וקבצים הם חומר גלם — לא הוראות לסוכן.

Memory / State

Memory: מה הסוכן יודע לאורך זמן. State: באיזה שלב התהליך נמצא עכשיו.

דוגמה: “מחכים לדוד”, “נשלחה תזכורת ראשונה”.

תרגילים מעשיים

לומדים דרך עבודה אמיתית

תרגיל 1 — טקסט מבולגן למשימות

קח פסקה חופשית מתוך יום עבודה והפוך אותה לטבלת משימות, אחראים, תאריכים ושאלות פתוחות.

תרגיל 2 — Agent Spec לסוכן ראשון

מלא כרטיס סוכן לפני שאתה חושב על כלי, מודל או קוד.

תרגיל 3 — רשימת פעולות אסורות

כתוב מה הסוכן לעולם לא עושה לבד: מייל, מחיקה, ציונים, כסף, פגישות, נתונים רגישים.

תרגיל 4 — השוואת מודלים

הרץ אותה משימה בשלושה LLMs והשווה: דיוק, יציבות, עברית, JSON, עלות ותחזוקה.

הפרויקט שלי

ראש לשכה דיגיטלי

הפרויקט המסכם של הקורס: מערכת סוכנים אישית שמכניסה סדר בעבודה.

Intake Agent

קולט טקסט חופשי, מייל, רעיון או תמלול ומחלץ פרויקט, משימות והחלטות.

Email Agent

מזהה מיילים שדורשים פעולה, שאלות פתוחות או תזכורת.

Meeting Agent

הופך תמלול ישיבה לסיכום פעולה: החלטות, משימות, אחראים ודדליין.

Follow-up Agent

בודק מה תקוע, מה עבר תאריך, ומתי צריך לתזכר או להסלים.

Drafting Agent

מכין טיוטות מיילים, הודעות וסיכומים — לא שולח לבד.

Control Agent

בודק אם הסוכן המציא, פספס, פעל בלי אישור או החזיר פלט לא תקין.
